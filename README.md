# md-note ベースライン定義

## 目的
自分専用のメモアプリを作成するための要件とベースライン（最初に合意する最小構成）を定義する。

## 要件
- メモは Markdown で記述できること。
- 通常の閲覧モードではレンダリングされた状態で閲覧できること。
- 編集ボタンで遷移できる編集モードで編集可能なこと。
- 一覧表示と検索ができること。
- Web、スマホアプリ、Windows アプリで動作すること。
- メモの内容は大した容量にならないため、維持コストを月 100 円以内に抑えること。

## ベースライン（最小構成）
### 1. 対応プラットフォーム
- Web（ブラウザ）
- スマホアプリ（iOS / Android）
- Windows アプリ

> 単一コードベースで複数プラットフォームに展開できる構成を前提にする。

### 2. 機能範囲（MVP）
- Markdown 編集
- レンダリング表示（閲覧モード）
- 編集ボタンで編集モードに遷移
- メモ一覧表示（タイトル/更新日時）
- 全文検索（タイトル + 本文）

### 3. データ保存とコスト
- 小容量を前提に、低コストの保存方式を採用する。
- 月 100 円以内の維持コストを厳守する。
- オフライン利用を前提に、クライアント側の保存/キャッシュを持つ。

### 4. 同期/バックアップ
- 低コストを優先しつつ、簡易同期またはバックアップが可能な構成にする。

### 5. セキュリティ
- 個人利用を前提とし、初期段階ではシンプルな構成を許容する。
- 将来の拡張に備え、認証追加が可能な設計にする。

## 追加で合意すべき項目
- 保存場所の優先順位（完全ローカル vs 低コストクラウド）
- 検索対象の範囲（タイトルのみ/本文含む）
- メモの構造（タイトル+本文/タグなどのメタ情報）

## 実装優先順位とバージョン分け
PoCとして使える最小構成を Version 0 とし、段階的に機能を拡張する。

### Version 0（PoC最小構成）
- 対応プラットフォーム: Web（ブラウザ）のみ
- Markdown 編集とレンダリング表示（閲覧モード）
- 編集ボタンで編集モードに遷移
- ローカル保存（ブラウザ内ストレージ）
- メモ一覧表示（タイトル/更新日時）
- 全文検索（タイトル + 本文）

### Version 1（MVP機能の完成）
- Markdownレンダリングの精度向上
- 一覧/検索の精度・UX改善
- 編集モード/閲覧モードの操作性向上

### Version 2（多プラットフォーム対応）
- スマホアプリ（iOS / Android）への展開
- Windows アプリへの展開
- オフライン利用の最適化

### Version 3（同期/バックアップと拡張性）
- 低コストでの同期/バックアップ
- 認証追加を見越した拡張性の確保

## Version 0 実装メモ
このリポジトリには Version 0 の PoC として、ブラウザで動作するシンプルな Markdown メモアプリを追加した。

### 使い方
1. このリポジトリをローカルに配置する。
2. `index.html` をブラウザで開く、または簡易サーバーで配信する。

```bash
python3 -m http.server 8000
```

ブラウザで `http://localhost:8000` を開くと利用できる。メモは `localStorage` に保存される。
